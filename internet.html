The modern internet evolved from a DOD project called ARPANet.

Network providers work with internet organizations:

  • ICANN is responsible for addresses on the internet.
  • The W3C is a governing body.
  • The IANA controls IP address allocation.

The internet is a network of connected networks. These 'sub-networks' are connected together in the internet infrastructure. Tier 1 networks are maintained by ISPs like Verizon, AT&T. Tier 1 networks form the internet backbone, the main routes for data.

Internet eXchange Points(IXP) are where ISPs exchange data.

IEEE (Institute of Electrical and Electronics Engineers) is a professional organizations that develops standards in engineering technology.

Ethernet technology is a wired network technology defined by IEEE 802.3 standards.

Standards for ethernet cabling define the speed of data transmission.

  • 10BaseT Ethernet, 802.3i,operates at 10 Mbps
  • Fast Ethernet, IEEE 802.3u, runs at 100 Mbps
  • Gigabit Ethernet, IEEE 802.3z can reach speeds of 1,000 Mbps
  • 10 Gigabit Ethernet, IEEE 802.3ae transfers data at a rate of 10,000 Mbps
  • 40/100 Gigabit Ethernet, IEEE 802.3ba, 40 to 100 Gbps

Your device should have an ethernet port, for connecting ethernet cabling. If not, an adpater can be used.

IP Version 6, IPV6, is an IP addressing schmeme that allows for 128 bits.

128 bits allows for 2128 IP addresses, approximately 340 undecillion.

We are still in transition to IP Version 6, IPV6.

IP Version 4, IPV4, is an IP addressing schmeme that allows for 32 bits.

32 bits allows for 232 IP addresses, approximately 4 billion.

We ran out of IPV4 addresses many years ago, although we are still in transition to IP Version 6, IPV6.

UDP (User Datagram Protocol) is an alternative to TCP. UDP also is responsible for 'chunking' data into packets and sending the packets out on the lines of the internet. UDP is faster than TCP, but UDP does not support any error checking!.

TCP (Transmission Control Protocol) is responsible for 'chunking' data into packets and sending packets out on the lines of the internet. If all packets are received, they are assembled in order. Otherwise, TCP requests a retransmission of any lost or corrupted packet.

World Wide Web Consortium, the governing body responsible for decisions about the future of the internet.

Each packet contains

  • the address of the sender
  • the destination address
  • a sequence number
  • data

The packets belonging to the original data may travel along different routes to get to the destination. When they arrive at the destination, they are reassembled in sequence with the sequence number.
