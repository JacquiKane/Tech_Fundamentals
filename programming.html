A programming language is any language software developers use to create programs, or applications. The code is termed source code, and must be converted to a form the computer can understand. This is done with tools called compilers and interpreters, into object code, and eventually executable code.

High Level Languages (HLL) include words like while, if, repeat that convey meaning. Low Level Languages typically have a basic instruction set only recognized by the processor. Assembler is a Low Level Language, whereas Java is considered a High Level Language.

Before any coding, developers often plan the code by writing Structured English plan of what the code will do. A form called pseudocode is closer to the language, but also a way of representing the coding. Another tool to plan the program design is Flowcharting, which uses a defined set of symbols.

Languages have evolved over the years and have categories that describe a level of closeness to a natural language, like English. First generation languages, 1GL and Second generation languages, 2GL, are closer to the machine, and as programming languages have evolved, third generation languages, 3GL, became more user friendly, like FORTRAN, and COBOL, C# and Java. Fourth generation languages, 4GL, are specialized towards particular domains, like database management. Examples include SQL and R. Fifth generation languages, 5GL, are used in artificial intelligence research and development.

Program flow depends on the design philosophy or paradigm used for developing the software. The procedural paradigm, reflects a series of steps and sequential execution of program code, whereas the Object Oriented Paradigm,OOP or Object Oriented Programming, uses communication between objects. These objects have properties and methods.

Specific languages support the procedural paradigm, or the object oriented paradigm, or both - depending on how the language is used by the developer.

There are 3 control structures, whatever the paradigm. These are sequence, selection and iteration.

Sometimes the same code is executed in different parts of a program. A function can contain these lines of code and be called with a label. A method acts the same way in OOP.

Object Oriented Programming, OOP is a program design strategy that uses Classes and Objects to build solutions to programming problems. A class is like a template for creating an object, and we say that an object has been newed up, or instantiated, from a class. Instance is the same as object.

Classes and objects have attributes and methods.

The 4 tenets of OOP are Abstraction, Polymorphism, Inheritance, and Encapsulation

Classes inherit from each other, saving code and design time. A class hierarchy is a set of classes that inherit from each other. The class used to derive another class is the superclass . The class that is derived from the superclass is the subclass . A UML diagram ,is used to describe classes in a hierarchy.
